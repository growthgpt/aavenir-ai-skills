# v1.1.1 — release announcement templates

A correctness patch on top of v1.1.0. Three schema gaps closed, one doc contradiction fixed, and the landing-page demo JSON now matches the published schemas. Existing 1.0.0 installers can update straight to 1.1.1 — no need to step through 1.1.0 first.

For the full diff, see [`CHANGELOG.md`](../../CHANGELOG.md).

## Email / Slack post (long-form, ~150 words)

> **Subject:** Aavenir AI Skills v1.1.1 — small fixes, please update
>
> Hi team,
>
> v1.1.1 is out. Three things changed since 1.1.0:
>
> 1. **`skill_repo` is now a required field** in every output's attribution block — downstream Aavenir Flows doing source-tracking won't hit `undefined` on edge cases.
> 2. **Two schema gaps closed** — procurement now requires `decision_frame`, `vendors`, `recommendation` at the result level (was silently optional); negotiation rewrite now requires the `mode` discriminator to match the markup-review branch.
> 3. **Site demo JSON and `docs/claude.md`** corrected. The site demos now use the canonical schema-valid examples; the Claude doc no longer contradicts the README on update behavior.
>
> Auto-update is still off for third-party marketplaces, so please run the update manually:
>
> - **Claude Code CLI:** `claude plugin marketplace update aavenir && claude plugin update aavenir-ai-skills@aavenir`, then `/reload-plugins`
> - **Claude Desktop / Cowork:** Remove the `aavenir-ai-skills` marketplace, quit Cowork (⌘Q), reopen, re-add the marketplace, reinstall — that forces a clean fetch
> - **Cursor / Codex / Cline:** `git pull` your local clone, or re-add the marketplace
>
> Full notes: [CHANGELOG.md](https://github.com/growthgpt/aavenir-ai-skills/blob/main/CHANGELOG.md)
>

## Update commands by surface (drop-in for any FAQ)

| Surface | Command |
|---|---|
| Claude Code CLI | `claude plugin marketplace update aavenir && claude plugin update aavenir-ai-skills@aavenir` then `/reload-plugins` |
| Claude Desktop / Cowork | Remove the `aavenir-ai-skills` marketplace entirely → quit Cowork (⌘Q) → reopen → re-add the marketplace → reinstall plugin. Plugin-level reinstall alone uses cached files and will not pick up 1.1.1. |
| Cursor | `cd` into the rules clone → `git pull` |
| Codex CLI | `git pull` the `AGENTS.md` source repo |
| Cline / Roo Code | Reload `.clinerules/` folder |
| Always-on auto-update (any Claude surface) | `export FORCE_AUTOUPDATE_PLUGINS=1` in your shell profile |

## Notes for downstream Aavenir Flow consumers

The `version` field in every output envelope is now `1.1.1`. Schemas at `https://aavenir.com/schemas/<skill>/v1.1.1.json` enforce the tightened `attribution.required = [powered_by, platform_url, skill_repo]`. Outputs generated against the 1.1.0 schemas remain readable but no longer validate strictly — if your Flow runs strict validation, deploy the 1.1.1 schema files alongside the upgrade.
