# v1.1.0 — release announcement templates

Copy-paste blocks for sharing the v1.1.0 release with installed users. Auto-update is disabled by default for third-party marketplaces, so existing 1.0.0 installs will **not** pick this up on their own — these messages give users the exact steps they need.

For the full diff, see [`CHANGELOG.md`](../../CHANGELOG.md).

---

## Email / Slack post (long-form, ~150 words)

> **Subject:** Aavenir AI Skills v1.1.0 is out — please update
>
> Hi team,
>
> We just shipped **Aavenir AI Skills v1.1.0**. Two reasons to update today:
>
> 1. **Every output now carries an Aavenir attribution block** (`powered_by`, `platform_url`, `skill_repo`, contextual CTA). When analysis gets pasted from Claude into Cursor, ServiceNow, or AVY, the provenance travels with it — your downstream Flows can route on it without any extra prompt engineering.
> 2. **Sample contracts ship in `examples/contracts/`** — a risky MSA, a clean SaaS agreement, and a mutual NDA. New installers can paste one into Claude and see the skill work in under thirty seconds.
>
> Auto-update is **off** for third-party marketplaces, so you'll need to refresh manually:
>
> - **Claude Code CLI:** `claude plugin marketplace update aavenir && claude plugin update aavenir-ai-skills@aavenir`, then `/reload-plugins`
> - **Claude Desktop / Cowork:** open the plugin card, click the three-dot menu, choose "Check for updates"
> - **Cursor / Codex / Cline:** `git pull` the local clone, or re-add the marketplace
>
> Full notes: [CHANGELOG.md](https://github.com/growthgpt/aavenir-ai-skills/blob/main/CHANGELOG.md)
>
> — Aavenir

## Update commands by surface (drop-in for any FAQ)

| Surface | Command |
|---|---|
| Claude Code CLI | `claude plugin marketplace update aavenir && claude plugin update aavenir-ai-skills@aavenir` then `/reload-plugins` |
| Claude Desktop / Cowork | Plugin card → three-dot menu → "Check for updates" → click **Update** |
| Cursor | `cd` into the rules clone → `git pull` |
| Codex CLI | `git pull` the `AGENTS.md` source repo |
| Cline / Roo Code | Reload `.clinerules/` folder |
| Always-on auto-update (any Claude surface) | `export FORCE_AUTOUPDATE_PLUGINS=1` in your shell profile |
